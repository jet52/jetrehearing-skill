# Petition for Rehearing — Recommendation Memo Format

## Complete Structure

```markdown
# PETITION FOR REHEARING — RECOMMENDATION MEMO

**Case No. {case_number}**
**{Case Name}**
**Opinion Date: {opinion_date}**
**Petition Filed: {petition_date}**
**Response: {Received / Not received}**
**Central Legal Staff**

## RECOMMENDATION

[¶1] **{Deny / Request response / Correction / Oral argument}.** {2-3 sentence
reasoning summarizing why this recommendation applies. Reference the Rule 40
standard — whether the petition identifies points of law or fact the Court
overlooked or misapprehended, or merely reargues the case.}

## RULE 40 COMPLIANCE

[¶2] {Timeliness: Was the petition filed within 14 days of the opinion under
N.D.R.App.P. 40(a)? Page limit: Does the petition comply with the 10-page
limit under N.D.R.App.P. 40(b)? Form: Does the petition comply with Rule 32
formatting requirements? Particularity: Does the petition "state with
particularity the points of law or fact that the petitioner believes the court
has overlooked or misapprehended" per N.D.R.App.P. 40(a)?}

## PETITION SUMMARY

[¶3] {Overview of the claims raised in the petition. Number of points claimed,
general thrust of the petition — does it argue overlooked law, overlooked facts,
misapprehended law, misapprehended facts, or a combination?}

## ISSUE-BY-ISSUE ANALYSIS

### Point 1: {Petition's characterization of the point}

[¶4] **Petition's Claim:** {What the petition argues the Court overlooked or
misapprehended, with pinpoint petition page cite.}

[¶5] **Opinion's Treatment:** {How the opinion actually addressed this point,
with pinpoint opinion paragraph cites [¶N]. State whether the opinion
addressed the point directly, addressed it implicitly, or did not address it.}

[¶6] **Record Support:** {Whether the petition's factual assertions are
supported by the record. **BOLD any failures**: "The petition cites R45:12
for the proposition that X, but **the cited page does not support this
characterization.**"}

[¶7] **Preservation:** {If original briefs were provided: whether this
argument was raised in the original briefing. Cite the brief page. If not
raised, note that rehearing is not a vehicle for raising new arguments.}

[¶8] **Assessment:** {Categorize: overlooked/misapprehended, mere
disagreement, or correction warranted. Explain why. If the petition merely
reframes the same argument in different words, say so.}

[¶9] **Response:** {If a response was filed: summarize the response's
argument on this point with pinpoint cite. If no response: omit this section.}

### Point 2: {Petition's characterization}

[¶10-...] {Same structure as Point 1.}

## NEW AUTHORITIES (if any)

{Table of citations in the petition that did not appear in the original
briefs or opinion. For each:}

| Citation | Cited For | In Original Briefs? | In Opinion? | Available at Time of Briefing? | Assessment |
|----------|-----------|---------------------|-------------|-------------------------------|------------|

{If the petition cites authorities not raised in the original briefing,
note that rehearing is not the proper vehicle for raising new authorities
that could have been raised earlier. If the authority is genuinely new
(e.g., decided after briefing), note that.}

## CONCLUSION

[¶last] {Restate the recommendation with summary reasoning. E.g.: "The
petition should be **denied**. The petitioner identifies no point of law or
fact that the Court overlooked or misapprehended. The petition reargues the
same positions presented in the original briefing, which the Court considered
and rejected."}
```

## Formatting Rules

### Paragraph Numbering
- Every paragraph is numbered: [¶1], [¶2], [¶3], etc.
- Numbering is **sequential throughout the entire memo** — do not restart at each section
- The RECOMMENDATION paragraph is always [¶1]
- CONCLUSION contains the final paragraph number

### Headings
- Major sections in **ALL CAPS**: `## RECOMMENDATION`, `## RULE 40 COMPLIANCE`, `## PETITION SUMMARY`, `## CONCLUSION`
- Issue headings use: `### Point N: {characterization}`
- NEW AUTHORITIES uses: `## NEW AUTHORITIES`

### Recommendation
- Always state a recommendation — this is not optional
- Use **bold** for the recommendation word in [¶1] and CONCLUSION
- Four possible recommendations:
  - **Deny** — Court did not overlook or misapprehend; petitioner merely disagrees
  - **Request response** — Close question on whether something was overlooked/misapprehended AND no response has been received yet
  - **Correction** — Petition identifies an error that should be fixed but does not change the result
  - **Oral argument** — Response has been received but the answer remains uncertain

### Content Requirements

#### RECOMMENDATION
- 1 paragraph
- Bold the recommendation word
- 2-3 sentences of reasoning
- Reference the Rule 40 standard

#### RULE 40 COMPLIANCE
- Check timeliness (14 days from opinion date)
- Check page limit (10 pages)
- Check form (Rule 32)
- Check particularity (N.D.R.App.P. 40(a))

#### ISSUE-BY-ISSUE ANALYSIS
Each point must include:
1. **Petition's Claim** — with petition page cite
2. **Opinion's Treatment** — with pinpoint [¶N] cites to the opinion
3. **Record Support** — **bold failures**
4. **Preservation** — if original briefs available
5. **Assessment** — categorize and explain
6. **Response** — if a response was filed

#### CONCLUSION
- 1-2 paragraphs
- Restate recommendation with bold
- Summary reasoning

### What to Avoid
- **Never** use placeholder brackets: [Date], [page], [County], etc.
- **Never** fabricate citations
- **Never** use "I" or "we"
- **Never** omit opinion paragraph cites when discussing the opinion's treatment
- **Never** add analysis beyond the Rule 40 standard (overlooked/misapprehended)
